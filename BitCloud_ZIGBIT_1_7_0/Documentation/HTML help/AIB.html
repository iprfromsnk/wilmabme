APS Information Base

This is a collection of variables that are passed between layers, in order to exchange information. These variables can be specified or obtained from a layer through its management service. The APS information base comprises the constants and attributes required to manage the APS layer of a device. This database may also include the access control list, which stores the security material.

The constants that define the characteristics of the APS sub-layer are presented below:

  • The maximum number of octets contained in a non-complex descriptor.
  • The maximum number of retries allowed after a transmission failure.
  • The maximum number of seconds to wait for an acknowledgement to a transmitted frame.
  • The minimum required size of the APS duplicate rejection table.
  • Fragmentation parameter - The maximum number of unacknowledged frames that can be active at once.
  • Fragmentation parameter - The standard delay between sending two blocks of a fragmented transmission.
  • The minimum number of octets added by the APS sub-layer to an ASDU.

The attributes of the AIB are listed below:

  • The current set of binding table entries in the device
  • TRUE if the device should become the ZigBee Coordinator on startup, FALSE if otherwise.
  • The mask of allowable channels for this device to use for network operations.
  • The 64-bit address of a network to form or to join.
  • The current set of group table entries.
  • The value to be used for the NonmemberRadius parameter when using NWK layer multicast.
  • The current set of permission configuration items.
  • A flag controlling the use of insecure join at startup.
  • Fragmentation parameter - the standard delay, in milliseconds, between sending two blocks of a fragmented transmission

Security-Related AIB Attributes:

  • A set of key-pair descriptors containing master and link key pairs shared with other devices.
  • Identifies the address of the device's Trust Center.
  • The period of time a device will wait for an expected security protocol frame (in milliseconds).
