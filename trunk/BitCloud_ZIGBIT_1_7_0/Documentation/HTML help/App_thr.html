Throughput Test Application

This application allows to perform measurement of maximum application data throughput in various transmission scenarios, such as with/without encryption, with/without APS level acknowledgments, with different payload sizes, etc. One coordinator and one router nodes are participating. Same firmware image should be used on both devices with role defined by DIP-switches configuration as follows: device type is detetmined by DIP switches of the MeshBean board as follows:

Device type DIP switches
SW4:1SW4:2SW4:3
CoordinatorONOFFOFF
RouterOFFONOFF

The network start procedure is initiated automatically on device startup. Green LED is periodically blinking with 500ms interval when device is joining the network and is ON if it is in the network.

Upon network join the router node after TIMER_DELAY_PERIOD starts to continuously send data packets over the air to coordinator. Packet transmission is indicated on the router by red LED. The coordinator receives data frames and measures the application level throughput based on number of payload bits received from the air and forwards this value to PC via USART once every TIMER_TRACK_PERIOD ms. Thus measured values can be observed in terminal window (e.g. Windows HyperTerminal). See COM port connection settings. Yellow LED is flashing when coordinator is receiving data and red LED is flashing when coordinator is forwarding measureds value to USART interface.

In order to measure maximum data throughput with encrypted data frames application shall be compiled with properly configured security-related parameters.

Important configuration parameters

APP_DEVICE_TYPESpecifies device type for platforms without DIP-switches.
APP_USE_APS_ACKSpecifies whether APS level acknowledgement (end-to-end) shall be requested for transmitted data frame.
TIMER_DELAY_PERIODTime interval (ms) after network join when coordinator starts measuring throughput and router starts sending frames.
TIMER_TRACK_PERIODTime interval (ms) used on coordinator for throughput measurement and output to serial interface.
CS_CHANNEL_MASKSpecifies channel mask to be used for network operation.
CS_CHANNEL_PAGESpecifies channel page to be used on SubGHz frequency band.
CS_EXT_PANIDSpecifies network's extended PANID. Should be unique in given location.
CS_UIDExtended address for platforms where it is not available via special UID chip or external EEPROM. If used CS_UID shall be added to CS_PARAMS list in application Makefile.

Note that description of parameters common for all applications can be found in Sample Applications overview. ConfigServer parameters are described in documentation of ConfigServer.h file.
